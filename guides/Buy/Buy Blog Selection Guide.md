# **Purpose**

This guide helps determine which Buy-stage blog type should be used for a content request.

At this stage, buyers have already:

```
✓ Understood the problem

✓ Explored solutions

✓ Evaluated approaches

✓ Built a business case

✓ Assessed readiness
```

Now they are attempting to reduce execution risk and build confidence.

---

# **Buy Stage Definition**

## **Buyer Mindset**

The buyer is moving toward a decision.

They are looking for:

```
Proof

Confidence

Validation

Risk Reduction

Practical Understanding
```

# **Decision Tree**

## **Step 1**

Is the content centered on a specific customer or organization?

### **Examples**

```
How XYZ Reduced Time-To-Hire

How ABC Built A Skills-Based Workforce

How DEF Improved Internal Mobility
```

If YES →

```
Customer Story
```

If NO →

Continue.

---

## **Step 2**

Is the content primarily focused on aggregated evidence, benchmarks, studies, or outcome data?

### **Examples**

```
Research Findings On Workforce Intelligence

The Results Organizations Achieve With Internal Mobility

What The Data Shows About Skills-Based Organizations
```

If YES →

```
Proof & Results
```

If NO →

Continue.

---

## **Step 3**

Is the content focused on the implementation journey and adoption process?

### **Examples**

```
How Organizations Implement Workforce Intelligence

Lessons From Internal Mobility Rollouts

Building An AI Governance Program
```

If YES →

```
Implementation Story
```

If NO →

Continue.

---

## **Step 4**

Is the content focused on a business scenario or workflow?

### **Examples**

```
Using Workforce Intelligence For Strategic Planning

How Agentic AI Supports Recruiters

Improving Internal Mobility With Skills Intelligence
```

If YES →

```
Use Case Deep Dive
```

If NO →

Continue.

---

## **Step 5**

Is the content focused on explaining a solution, capability, platform, or offering?

### **Examples**

```
What Is Workforce Intelligence?

Understanding Talent Intelligence

Introducing Agentic Recruiting
```

If YES →

```
Solution Spotlight
```

# **Blog Type Summaries**

---

## **Customer Story**

### **Core Question**

```
Has someone like us succeeded?
```

### **Purpose**

Demonstrate success through a real organization.

### **Reader Takeaway**

```
Organizations similar to ours have solved this challenge.
```

### **Typical Outputs**

```
Case Study

Success Story

Customer Journey

Transformation Story
```

---

## **Use Case Deep Dive**

### **Core Question**

```
How would this work for us?
```

### **Purpose**

Show how a business scenario operates in practice.

### **Reader Takeaway**

```
I can visualize this in my environment.
```

### **Typical Outputs**

```
Workflow Guide

Business Scenario

Operational Example

Persona-Based Use Case
```

---

## **Solution Spotlight**

### **Core Question**

```
What exactly does the solution do?
```

### **Purpose**

Explain a solution and its capabilities.

### **Reader Takeaway**

```
I understand the solution and why it matters.
```

### **Typical Outputs**

```
Solution Overview

Capability Overview

Platform Introduction

Product Education
```

---

## **Implementation Story**

### **Core Question**

```
What happens after adoption?
```

### **Purpose**

Show the journey from decision to adoption.

### **Reader Takeaway**

```
I understand what execution and change management involve.
```

### **Typical Outputs**

```
Implementation Journey

Adoption Story

Transformation Rollout

Program Launch
```

---

## **Proof & Results**

### **Core Question**

```
What evidence supports success?
```

### **Purpose**

Provide aggregated proof and measurable outcomes.

### **Reader Takeaway**

```
There is credible evidence that this approach works.
```

### **Typical Outputs**

```
Research Summary

Outcome Analysis

Benchmark Findings

Evidence Report
```

# **Common Classification Mistakes**

---

## **Mistake \#1**

### **Customer Story vs Implementation Story**

Bad Classification:

```
How XYZ Launched A Workforce Intelligence Program
        ↓
Customer Story
```

Potentially Correct:

```
Implementation Story
```

Reason:

The focus is on adoption and rollout.

Not primarily outcomes.

---

## **Mistake \#2**

### **Customer Story vs Proof & Results**

Bad Classification:

```
The Results Organizations Achieve With Skills Intelligence
        ↓
Customer Story
```

Correct:

```
Proof & Results
```

Reason:

Evidence is aggregated.

Not tied to one organization.

---

## **Mistake \#3**

### **Use Case Deep Dive vs Solution Spotlight**

Bad Classification:

```
How Recruiters Use Agentic AI To Prioritize Candidates
        ↓
Solution Spotlight
```

Correct:

```
Use Case Deep Dive
```

Reason:

The focus is workflow application.

Not the solution itself.

---

## **Mistake \#4**

### **Solution Spotlight vs Use Case Deep Dive**

Bad Classification:

```
What Is Workforce Intelligence?
        ↓
Use Case Deep Dive
```

Correct:

```
Solution Spotlight
```

Reason:

The article explains the solution category.

Not a scenario.

# **Routing Priority**

When multiple templates appear relevant:

```
1. Customer Story

2. Proof & Results

3. Implementation Story

4. Use Case Deep Dive

5. Solution Spotlight
```

---

# **Why This Priority Exists**

## **Customer Story**

Usually easiest to identify.

Contains named organizations and outcomes.

---

## **Proof & Results**

Strong evidence-driven structure.

Distinct from narrative content.

---

## **Implementation Story**

Has a unique adoption and execution focus.

---

## **Use Case Deep Dive**

Focused on workflows and scenarios.

---

## **Solution Spotlight**

Acts as the default explanation-oriented Buy template.
