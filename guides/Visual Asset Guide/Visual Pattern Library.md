# **Visual Pattern Library**

**Version:** V2.0  
**Purpose:** Knowledge Guide  
**Stage:** Visual Strategy → Visual Asset Specification

---

# **Overview**

The Visual Pattern Library defines the approved visual communication patterns that may be recommended for content assets.

Each pattern exists to solve a specific communication problem. Visual patterns should be selected based on the structure of information, the reader's needs, and the communication objective—not personal preference or aesthetics.

This guide standardizes the visual language used across all content to ensure consistency, clarity, and scalability.

This guide does **not** determine whether a visual is required. That decision is made in the **Visual Decision Framework**.

---

# **Guiding Principles**

Every recommended visual should:

* Improve reader comprehension  
* Reduce cognitive load  
* Simplify complex information  
* Support faster understanding  
* Reinforce key messages  
* Improve scanability  
* Present information with clarity

Visuals should never exist purely for decoration.

---

# **Pattern Categories**

Visual patterns fall into two categories.

## **Editorial Assets**

Editorial assets establish context and visual identity.

| Pattern | Purpose |
| ----- | ----- |
| Hero Image | Introduce the article and establish tone |

---

## **Informational Assets**

Informational assets explain, organize or summarize information.

| Pattern | Purpose |
| ----- | ----- |
| Process Diagram | Explain workflows |
| Framework Diagram | Explain conceptual models |
| Timeline | Show progression over time |
| Comparison Matrix | Compare alternatives |
| Decision Tree | Guide decisions |
| Data Visualization | Present research |
| Architecture Diagram | Explain systems |
| Journey Map | Explain experiences |
| Roadmap | Show phased implementation |
| Checklist Graphic | Summarize actions |
| KPI Dashboard | Present measurable outcomes |
| Infographic | Combine multiple visual narratives |

# **Pattern 00 — Hero Image**

## **Purpose**

Introduce the article and establish its editorial identity.

The Hero Image should create immediate context while encouraging the reader to engage with the article.

---

## **Communication Objectives**

* Introduce  
* Inspire  
* Contextualize  
* Humanize

---

## **Best For**

* Blog articles  
* Editorials  
* Thought Leadership  
* Research articles  
* Guides  
* Industry perspectives

---

## **Typical Components**

* Primary subject  
* Environmental context  
* Editorial composition  
* Visual metaphor  
* Clear focal point  
* Negative space for title overlays

---

## **Should Not Include**

* Diagrams  
* Charts  
* Process flows  
* Tables  
* Large amounts of text  
* UI screenshots (unless central to the article)

---

## **Information Density**

Low

---

## **Reader Goal**

Generate curiosity and establish context.

# **Pattern Selection Rules**

The AI should:

* Recommend one Hero Image for editorial content where appropriate.  
* Recommend one primary supporting visual for most articles.  
* Recommend additional supporting visuals only when they improve comprehension.  
* Avoid recommending multiple visuals that communicate identical information.  
* Select the simplest pattern capable of communicating the intended message.  
* Maintain consistency with the communication objective identified in the Visual Decision Framework.

---

# **Output Format**

Every recommendation should produce a standardized **Visual Pattern Specification**.

```
Hero Image

Required: Yes

Purpose:
Introduce the topic and establish editorial context.

Supporting Visual

Recommended Pattern:
Framework Diagram

Communication Objective:
Visualize

Information Structure:
Layered

Information Density:
Medium

Reader Goal:
Understand relationships between five strategic concepts.

Typical Components:
- Central concept
- Supporting pillars
- Directional relationships
- Concise labels

Recommended Placement:
After Introduction

Confidence:
High

Reason:
The article introduces a conceptual framework that is more easily understood through a structured visual representation.
```

---

## **Pattern Selection Principles**

When multiple patterns appear suitable, prioritize them in the following order:

1. **Reader comprehension** over visual sophistication.  
2. **Information structure** over visual preference.  
3. **Communication objective** over aesthetics.  
4. **One strong visual** over several redundant visuals.  
5. **Simplicity** over unnecessary complexity.  
6. **Consistency** with established visual patterns across the content library.
