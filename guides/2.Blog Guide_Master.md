# **Blog Guide (Master)**

Version: 1.0

## **Purpose**

This guide serves as the source of truth for all blog content produced within the Eightfold Content System.

It defines:

* When a blog should be used  
* How blogs are classified  
* How blogs align with the Buyer Journey Framework  
* How blog templates are selected  
* Metadata standards  
* Drafting standards  
* Review standards

This guide does not define specific blog structures.

Blog structures are defined in individual Blog Type Guides.

---

# **What Is A Blog?**

A blog is a content asset designed to answer a specific audience question.

Blogs are primarily used to:

* Educate  
* Build credibility  
* Explain concepts  
* Reframe industry thinking  
* Support evaluation  
* Enable buyer progression

Blogs should always be:

* Persona-driven  
* Stage-aware  
* Evidence-based  
* CTA-oriented

---

# **Blog Classification System**

Every blog must be classified using three dimensions.

## **Layer 1: Content Type**

```
Content Type: Blog
```

Fixed value.

## **Layer 3: Buyer Stage**

Defines:

Why the content exists.

A blog must be assigned exactly one stage.

```
Awareness
Discovery
Consideration
Buy
```

Buyer stage must be determined using:

* Buyer Journey Framework  
* Customer Lens  
* Stage Question  
* Eightfold Lens

Never infer stage solely from topic.

---

# **Blog Routing Logic**

Use this sequence.

## **Step 1**

Identify the audience question.

Example:

```
Why are recruiters overwhelmed?
```

---

## **Step 2**

Identify buyer stage.

Example:

```
Reader is naming the problem

→ Awareness
```

---

## **Step 3**

Identify blog type.

Example:

```
Explaining the problem

→ Problem Identification
```

---

## **Result**

```
Content Type: Blog

Blog Type: Problem Identification

Buyer Stage: Awareness
```

---

# **Prohibited Content**

Never:

* Invent product capabilities  
* Invent customer outcomes  
* Invent statistics  
* Invent analyst findings  
* Invent compliance claims  
* Invent customer quotes

Flag missing information for review.

---

# **Drafting Principles**

Every blog should:

## **Answer A Question**

Readers should immediately understand:

```
What question is this article answering?
```

---

## **Teach Before Selling**

Educational value should come before product discussion.

---

## **Support Claims**

Every significant claim should be traceable to an approved source.

---

## **Maintain Stage Discipline**

Do not move prematurely into:

* Product positioning  
* Competitive comparisons  
* Implementation details

unless the assigned stage requires it.

---
