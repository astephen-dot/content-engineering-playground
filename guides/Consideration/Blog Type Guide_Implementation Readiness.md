# **Purpose**

An Implementation Readiness blog helps organizations determine whether they are prepared to successfully adopt a new solution, initiative, or operating model.

The goal is not to explain implementation steps.

The goal is not to provide a project plan.

The goal is to help readers assess:

* Organizational readiness  
* Capability gaps  
* Risk areas  
* Stakeholder alignment  
* Success prerequisites

The reader should leave saying:

"We understand what needs to be in place before moving forward."

# **Audience Mindset**

The audience has already determined:

```
This approach looks promising.
```

Now they are asking:

```
Are we ready?

What could derail success?

What capabilities do we need first?

What should we prepare for?
```

---

# **Content Objective**

The article should:

✓ Assess readiness

✓ Highlight success factors

✓ Identify capability gaps

✓ Surface risks

✓ Help organizations prepare

The article should NOT:

✗ Provide implementation instructions

✗ Recommend vendors

✗ Focus on technical configuration

✗ Create project plans

✗ Assume readiness exists

# **Readiness Integrity Rules**

## **Avoid Implementation Detail**

Bad:

```
Step 1: Configure the platform.
```

Good:

```
Ensure required capabilities and resources are available.
```

---

## **Focus On Readiness, Not Execution**

Ask:

```
Are we prepared?
```

Not:

```
How do we deploy this?
```

---

## **Readiness Is Multi-Dimensional**

Always assess:

```
Strategy

People

Process

Data

Technology

Governance
```
