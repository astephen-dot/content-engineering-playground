# **Discovery Blog Selection Guide**

Version: 1.0

---

# **Purpose**

This guide helps writers, editors, AI systems, and workflow automations select the correct **Discovery Blog Type** before content creation begins.

The objective is to ensure content aligns with the reader's intent during the **Discovery stage**, where audiences move from understanding a problem to exploring potential approaches, concepts, and strategic directions.

---

# **Discovery Stage Definition**

## **Reader Mindset**

At the Discovery stage, readers have already recognized a challenge or opportunity.

They are now trying to understand:

```
How are organizations addressing this?

What approaches exist?

What should I learn more about?

What should I evaluate further?
```

---

## **Content Goals**

Discovery content should help readers:

* Understand solution categories  
* Learn concepts and terminology  
* Explore strategic approaches  
* Learn from experts and peers  
* Understand best practices

Discovery content should NOT focus on:

* Vendor selection  
* Product comparison  
* Pricing  
* Implementation projects  
* Purchase decisions

---

# **Discovery Blog Decision Tree**

## **Question 1**

Is the article primarily explaining a concept, methodology, or technology?

### **YES**

Use:

→ Explainer

### **Example**

```
What Is Workforce Intelligence?

How Agentic AI Works In Recruiting
```

---

## **Question 2**

Is the article introducing a structured model or way of thinking?

### **YES**

Use:

→ Framework

### **Example**

```
A Framework For Responsible AI Governance

The Four Pillars Of Workforce Agility
```

---

## **Question 3**

Is the article exploring different approaches organizations use to solve a challenge?

### **YES**

Use:

→ Solution Exploration

### **Example**

```
How Organizations Are Addressing Workforce Visibility Challenges
```

---

## **Question 4**

Is the article focused on lessons learned from successful organizations?

### **YES**

Use:

→ Best Practices

### **Example**

```
5 Best Practices For Internal Mobility Programs
```

---

## **Question 5**

Is the primary source material a webinar, event, panel, or roundtable?

### **YES**

Use:

→ Webinar Follow-Up

### **Example**

```
Key Takeaways From Our AI Governance Webinar
```

---

# **Selection Matrix**

# **Quick Reference Guide**

## **Explainer**

### **Purpose**

Build understanding.

### **Starts With**

```
What is this?
```

### **Reader Takeaway**

```
I understand the concept now.
```

### **Common Triggers**

* Definitions  
* Concepts  
* Methodologies  
* Technologies  
* Business practices

---

## **Framework**

### **Purpose**

Provide a structured model.

### **Starts With**

```
Here's a way to think about it.
```

### **Reader Takeaway**

```
I have a framework for evaluating this challenge.
```

### **Common Triggers**

* Maturity models  
* Pillars  
* Dimensions  
* Frameworks  
* Strategic models

---

## **Solution Exploration**

### **Purpose**

Explore approaches.

### **Starts With**

```
Organizations are solving this in different ways.
```

### **Reader Takeaway**

```
I understand the available approaches.
```

### **Common Triggers**

* Solution categories  
* Strategic approaches  
* Emerging practices  
* Alternative methods

---

## **Best Practices**

### **Purpose**

Highlight proven behaviors.

### **Starts With**

```
Successful organizations consistently do these things.
```

### **Reader Takeaway**

```
I know which practices matter most.
```

### **Common Triggers**

* Lessons learned  
* Recommendations  
* Success patterns  
* Organizational behaviors

---

## **Webinar Follow-Up**

### **Purpose**

Extract insights from expert discussions.

### **Starts With**

```
Experts recently discussed...
```

### **Reader Takeaway**

```
I understand the key insights from the discussion.
```

### **Common Triggers**

* Webinar recordings  
* Roundtables  
* Panels  
* Virtual events  
* Conference sessions

# **Similar-Looking Requests (Common Routing Mistakes)**

---

## **Explainer vs Framework**

### **Explainer**

```
What Is Responsible AI?
```

Goal:

```
Understand the concept.
```

---

### **Framework**

```
A Framework For Responsible AI Governance
```

Goal:

```
Organize and evaluate the concept.
```

---

## **Solution Exploration vs Best Practices**

### **Solution Exploration**

```
How Organizations Are Improving Internal Mobility
```

Goal:

```
Explore multiple approaches.
```

---

### **Best Practices**

```
5 Best Practices For Internal Mobility Programs
```

Goal:

```
Identify proven behaviors.
```

---

## **Webinar Follow-Up vs Best Practices**

### **Webinar Follow-Up**

```
Key Takeaways From Our Workforce Intelligence Webinar
```

Goal:

```
Extract insights from an event.
```

---

### **Best Practices**

```
Best Practices For Workforce Intelligence Programs
```

Goal:

```
Teach lessons independent of the webinar.
```

# **AI Routing Rules**

When multiple blog types seem possible, use the following priority order:

```
1. Webinar Follow-Up
2. Framework
3. Best Practices
4. Explainer
5. Solution Exploration
```

### **Why?**

Because:

* Webinar source material determines structure.  
* Frameworks have a unique architecture.  
* Best Practices have a distinct list-based pattern.  
* Explainers answer explicit educational questions.  
* Solution Exploration becomes the default when the goal is comparing approaches.

| Extract insights from an event | Webinar Follow-Up |
